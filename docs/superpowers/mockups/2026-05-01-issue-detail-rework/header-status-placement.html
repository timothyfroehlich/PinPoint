Pros
  • Status gets dedicated, visible real estate
  • "Stop and notice" weight — good for technicians scanning urgency
  • Long status names breathe (own row on mobile if needed)
Cons
  • On mobile, status competes with back link for top-right attention
  • Two visual weights for "metadata" (hero + pills) is harder to learn

My read: Variant A (Linear) is the more disciplined choice — one pill vocabulary across all metadata, no special "hero" exception, easier to teach. The status pill earns its prominence through position (start of the row) rather than size, which is cleaner for the design system. Variant B's hero treatment is louder but creates a "this one field is special" pattern we'd have to remember in every other detail page (machine, location).
