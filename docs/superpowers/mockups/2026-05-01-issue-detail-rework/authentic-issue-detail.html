Corrections from earlier mockups

Built from src/server/db/schema.ts issues table, src/lib/issues/status.ts color tables, and src/lib/permissions/matrix.ts access rules. All field names, values, colors, and editability match the live code.
Added (was missing)
  • Severity pill (cosmetic / minor / major / unplayable, amber gradient, AlertTriangle icon)
  • Frequency pill (intermittent / frequent / constant, cyan gradient, Repeat icon)
  • OwnerRequirementsCallout — appears between metadata and description when machine has owner requirements set
  • Description as rich text (ProseMirror jsonb) — supports paragraphs, formatting
  • Real status palette — 11 statuses with per-status colors (sky for In Progress, green for Fixed, etc.) — not a generic blue
  • Permission tier split — title/status/severity/frequency = issues.update.reporting (guest can edit own); priority/assignee = issues.update.triage (member+ only)
Removed (didn't exist)
  • Location pill — no locations table in schema
  • Editable Machine pill — machine is fixed at issue creation; URL identity, never re-assignable
Fixed
  • Pill colors follow the existing field-family system (severity=amber, priority=purple, frequency=cyan) — not generic gray
  • Reporter modes — schema has 4 mutually-exclusive modes (member, invited, anonymous-name/email, none); display name only per email-privacy rule
  • Game Owner — link only present when the machine has an owner; reporter gets an "Owner" flag when reporter IS the machine owner
  • Issue ID badge — uses formatIssueId(initials, issueNumber) pattern (e.g., PIN-101)
Editable fields — final list (4 pills + 1 hero):
  • Status (hero pill, top-right) → grouped popover (Open/In Progress/Closed) with 11 options, tooltips, per-status icon+color
  • Priority (pill) → 3 options, purple gradient, TrendingUp icon — member+ only
  • Severity (pill) → 4 options, amber gradient, AlertTriangle icon
  • Frequency (pill) → 3 options, cyan gradient, Repeat icon
  • Assignee (pill) → searchable list of members + admins (no guests) — member+ only
Open question for next round: Status hero is sized larger than the other pills (5px vs 4px padding, 0.75rem vs 0.7rem font). Does that read as "appropriately prominent" or as "the inconsistency I warned about earlier"? If the latter, we make status a regular pill in the same row as the other 4, just placed first or last for prominence.
