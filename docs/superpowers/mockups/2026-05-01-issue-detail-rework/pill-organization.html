Pill row organization

Three ways to break up the flat 8-pill row. Same data, different organization. Pick the structure that gives you the cleanest scanning experience.

α Reduce: 3 editable pills, push the rest into header chrome

Pull identity context (machine, location) into a small eyebrow above the title. Pull engagement signals (reporter, created, watchers) into a footer subtitle line. The pill row becomes only what you actually edit: Priority, Assignee, Machine.

Pros
  • Pill row earns its visual weight — only edit-frequent fields
  • Identity/context lives in the title area where it's read first
  • Cleanest mobile (3 pills wrap nicely)
  • Closest to Linear's actual practice
Cons
  • Header is denser — eyebrow + title + status + subtitle
  • Watchers count is small in the subtitle, not pill-sized
  • "More" panel becomes redundant — everything's on screen

β Group: keep all 8 pills, organize by role

Same pill row, but split into semantic groups with vertical dividers. Group 1 = state (priority). Group 2 = people (assignee). Group 3 = context (machine, location). Group 4 = engagement (watchers, reporter, created, watch).

Pros
  • Nothing's hidden — every field visible
  • Scanning by group is faster than by individual pill
  • Same vocabulary at every viewport
Cons
  • Dividers add visual chrome the design bible hasn't defined
  • Mobile wrap breaks groups across lines awkwardly
  • Still 8 pills — high cognitive load even with organization

γ Two-tier: editable row above, static row below

Tier 1 (filled, prominent) holds the editable pills. Tier 2 (transparent, lighter) holds the static reference data. Visually distinct; tier 1 is where you act, tier 2 is what you read.

Pros
  • Strong visual hierarchy: action vs reference
  • Tier 2 can wrap independently without confusing tier 1
  • Anonymous user mode is natural — tier 1 just goes static (still distinct from tier 2)
Cons
  • Two pill styles to maintain in the design system
  • Vertical space cost — adds a row
  • Boundary between "editable" and "static" can feel arbitrary (machine: editable but feels like context)

My recommendation: α. The right move is fewer pills, not more organization for too many. A 3-pill row (Priority / Assignee / Machine) reads as "the things you change about this issue" — that's a clean concept. Identity (location, machine) belongs in the title eyebrow because that's how you contextually find the issue. Engagement (watchers, reporter, created) belongs in the subtitle because it's about people and time, not state. β just lipsticks the pig (8 pills is still 8 pills). γ is structurally sound but pays a vertical-space tax for a real-estate problem α solves by relocating, not stacking.
