How the responsive flip works (educational note):
  • .pp-rows-container sets container-type: inline-size — declares that descendants can query its width
  • Inside @container (min-width: 40rem), the grid switches from 1 column to 2 columns and the Assignee row spans both columns
  • Below that breakpoint (mobile, narrow side panels, etc.) — single-column stack
  • The 40rem threshold is tunable. Could go to 32rem (more aggressive 2-col on tablets) or 48rem (more conservative)

Why container query, not viewport breakpoint: if we ever reuse this metadata block inside a narrower context (e.g., a comparison view, or a side panel), it'll automatically fall back to 1-col without any code change. Viewport breakpoints would be wrong about width whenever the component isn't in its "main column" context.
