## What it does

Discord powers two features in PinPoint:

- **Sign in with Discord** — once enabled by an admin, members can authenticate with their Discord account instead of (or in addition to) an email and password.
- **Direct-message notifications** — members who connect their Discord account can receive issue updates as DMs from the PinPoint bot, alongside (or instead of) email and in-app notifications.

Both features depend on the same admin-side configuration. They can be set up independently per user from **Settings**.

## How users connect their account

1. Go to **Settings → Connected Accounts → Connect Discord** and authorize PinPoint.
2. Use **Send test DM** under the Discord row to verify the bot can reach your inbox.
3. Open **Settings → Notifications** and pick which events you want as DMs (Discord defaults match your email selections).

You can disconnect Discord at any time from the same Connected Accounts row — you'll be asked to confirm. Disconnecting stops DMs immediately; re-link to resume.

## How admins enable it

1. Go to **Integrations → Discord** (the Integrations link is in the user dropdown, admins only).
2. Paste the bot token from the Discord developer portal and save it.
3. Optionally set the **Guild ID** for the Discord server and an **Invite link** — the invite link is surfaced to users when DMs fail because they aren't in the server.
4. Flip the **Integration enabled** switch. The switch stays disabled until a token has been saved.
5. Use **Verify bot token** under Test connection to confirm the bot can authenticate against Discord.

Without a saved token the "Sign in with Discord" button stays hidden for everyone, and Discord is omitted from each user's notification preferences.

## Troubleshooting

- **"Sign in with Discord" button is missing on the login page.** An admin hasn't enabled the integration yet. Ask an admin to set the bot token and flip the Enabled switch.
- **DMs aren't arriving.** The bot can only DM users who share a server with it. Make sure you're in the configured Discord server — admins can share the invite link from the Integrations page. If you've blocked DMs from server members in your Discord privacy settings, allow them.
- **"Send test DM" returns "blocked".** Same root cause as above — usually privacy settings or not sharing a server with the bot.
- **Bot health shows anything other than "healthy".** Possible values are `unknown`, `healthy`, and `degraded`. Re-check the bot token under Integrations and make sure the bot user hasn't been removed from the server.
